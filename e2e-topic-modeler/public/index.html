How It Works

Five steps matching the dissertation methodology — no coding required.

1

Embed

Documents are converted into dense 1024-dimensional semantic vectors using Voyage AI (voyage-3), Anthropic's recommended embedding model.

2

Reduce

UMAP (Uniform Manifold Approximation and Projection) reduces the high-dimensional embeddings to a lower-dimensional space using the cosine metric.

3

Cluster

HDBSCAN discovers clusters of varying density in the UMAP-reduced space, using mutual reachability distance and the Excess of Mass (EOM) extraction method.

4

Name

Representative documents from each cluster are sent to Claude 5 times. The topic name receiving the most "votes" wins — a democratic LLM naming process.

5

Classify

Every document in the corpus is classified into one of the discovered topics by Claude, including noise points not assigned during clustering.

About E2E Topic Modeling

The dissertation methodology, made accessible through the browser.

Embedding-to-Explanation (E2E) Topic Modeling is a hybrid analytic approach in which documents are embedded into dense semantic space, reduced via UMAP, and clustered with HDBSCAN to discover topical structure. The statistically derived clusters are then interpreted by Claude through democratic LLM voting to produce human-readable topic names and document classifications.

The pipeline matches the dissertation methodology:

  • Voyage AI (voyage-3) — dense document embeddings
  • UMAP — dimensionality reduction (n_neighbors=15, n_components=5, min_dist=0.01, cosine)
  • HDBSCAN — density-based clustering with EOM cluster extraction
  • Claude (democratic voting) — topic naming with 5-vote majority consensus
  • Claude (classification) — document-level theme assignment

UMAP and HDBSCAN run locally in your browser on the embedding vectors. The embedding, topic-naming, and classification stages communicate with a Cloudflare Worker that securely proxies requests to Voyage AI and Claude.

Your CSV must include an id column and a text column. Additional columns are preserved and included in the results download. The file should be UTF-8 encoded.
